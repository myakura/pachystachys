HTMLって

どんなものだっけ

HyperText Markup Language

  • 「ハイパーテキスト」の言語
    （リンクやメディアが入る）
  • 「ハイパー」だけど「テキスト」

→ 基本的にはテキスト。

基本的にはテキスト…？

可搬性が（まあまあ）高い。

  • 古いブラウザでも読める※
  • エディタでも読める※
  • 非ブラウザでも処理できる ※

→（まあまあ）扱いやすい。

情報伝達に優れている

  • 可搬性が高い→加工しやすい
  • 特定のソフトに囚われにくい

PDFやWordに比べ、伝えやすい。

伝えやすさは価値。

HTMLで
構造化する

伝えやすさを高めるために

文書の構造をちゃんと記す。

  • 見出し（<h1>-<h6>）
  • 段落（<p>）
  • リスト（<ul>, <ol>, <dl>）

HTML5でさらに詳細に

  • セクション関連要素の導入
  • 新しいブロック要素の追加
  • 新しいインライン要素の追加

今までよりも細かく表現できる。
加工時のフックも増える。

セクション関連要素

  • <section> ― 汎用セクション
  • <nav> ― 主要なナビゲーション
  • <aside> ― 補足、サイドバー
  • <article> ― 独立可能なもの

<section>

文章のなかのひとまとまり。 

<section id="section-element">
  <h2>section要素</h2>
  <p>section要素は章や節など、文章における一般的な...
  <p>sectionはdivではありません。見た目のために...
</section>

※ 見た目のためなら<div>を使おう

<nav>

グローバルナビとか、目次とか。 

<nav id="toc">
  <h2>目次</h2>
  <ol>
    <li><a ...>section要素</a></li>
    <li><a ...>nav要素</a></li>
    <li><a ...>aside要素</a></li>
    <li><a ...>article要素</a></li>
  </ol>
</nav>

<aside>

本文とそんなに関係ないものに。 

<section id="nav-element">
  <h2>nav要素</h2>
  ...
  <aside>
    <h3>「navi」ではなく「nav」</h3>
    ...
  </aside>
</section>

<article>

<h1>過去のエントリ：2014年3月</h1>
<article class="entry" ...>
  <h1>3/8 仙台！</h1>
  ...
</article>
<article class="entry" ...>
  <h1>3/10 びーる！</h1>
  ...
</article>

<header>...</header>

いろいろできるCSSレイアウト仕様。

UIを作るのに特に便利になる。

表示上の順番を入れ替えられたりも。

dev.w3.org/csswg/css-flexbox/

例：html5jのWebサイト

<section>
  <a ...>
    <h4>味見部</h4>
    <img ...> <!-- ソース上は画像が後 -->
  </a>
  <p>真新しい技術を実際に触ってみることを目的と...
</section>

orderプロパティで入れ替え

a {
  display: flex;
  flex-direction: column;
}
a img {
  order: -1; /* 見た目上<h4>の上に */
}
